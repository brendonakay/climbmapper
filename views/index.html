About Climb Mapper

This application is an experiment to visualize a climbers climbing past and future from a geographic perspective. Specifically, it displays climbing areas sized by number of Ticks and ToDos for each area. Charts are used throughout to display the breakdown of climb rating (using YDS) or type (ex: boulder, sport, traditionsl) by area. All the scrapy code is available on one of my GitHub repos.

About The Data

All route data (Ticks and ToDos) came from my user account on Mountain Project. All climbing area data is created by me independent of Mountain Project.

About The Technology Used

The application is built on PostgreSQL/PostGIS, Node.js, Leaflet.js, D3.js, Bootstrap, and JQuery. Needless to say there is a lot of Javascript used for presentation and a little Pyhon for data extraction/processing.

Future Plans

Because this is an experiment you can expect the visualizations to get better and more interesting. Specifically I want to add:

  1. ability for others to add area points (that data would be shared back to the community)
  2. different map layer point symbol types
  3. better charts
  4. a better time slider design
  5. better layout and behavior when clicking on area points
  6. generally better page design and layout
I would also like to add the ability for other Mountain Project users to visualize their info but a lot of work would need to be done before that is possible.
